🤔 What is NetSpy?

NetSpy is a Chrome DevTools extension designed for frontend developers, providing powerful network request interception, modification, and debugging capabilities.

Core Capabilities

  • Real-time Interception: Pause requests before sending or responses before returning to the page
  • Request Modification: Modify URL, Headers, Body and resend
  • Response Modification: Change status code, headers, and body before page receives it
  • Mock Data: Return custom mock responses without backend involvement
  • Request Replay: Resend any request with one click for easy debugging

How is it Different from Chrome Network Panel?

  • Chrome Network can only view requests; NetSpy can intercept and modify
  • Chrome Network cannot mock responses; NetSpy can return any custom data
  • NetSpy provides a friendlier editing interface with JSON formatting, syntax highlighting, etc.

🚀 Getting Started

Open Chrome DevTools
Press F12 or right-click the page and select "Inspect"
Find the NetSpy Panel
Look for the NetSpy tab in the DevTools top tab bar and click it
Enable Interception
Toggle the Intercept switch and select a mode (Request/Response/Both)
Set URL Pattern
Enter a URL pattern like * (all) or */api/* (API only)
Trigger Requests
Refresh the page or interact to trigger network requests; matching ones will be paused
Handle Paused Requests
Click Edit, Mock, Continue, or Drop to handle the paused request

🎯 Interception Modes

Request Request Mode

Pauses requests before they're sent to the server. Use cases:

  • Modify request parameters (URL, Query, Headers, Body)
  • Test how different request data affects the backend
  • Block certain requests from sending (Drop)
  • Return mock data without hitting the real backend
💡 Tip
In this mode, you can use the Mock feature to return custom responses without actually sending the request.
Response Response Mode

Pauses responses before they reach the page. Use cases:

  • Modify data returned by the backend (status code, headers, body)
  • Simulate error scenarios (500, 404, timeout, etc.)
  • Test frontend handling of different response data
  • Temporarily modify API responses without changing backend code
ℹ️ Note
Response interception lets the request send normally, then pauses when the backend responds so you can modify it.
Both Both Mode

Intercept both requests and responses for complete control:

  • Modify request parameters first, then modify the response after it returns
  • Ideal for scenarios requiring full control over the request-response cycle
  • Note: Each request will pause twice (request phase + response phase)

💼 Use Cases

🐛 Scenario 1: Debug API Request Parameters

Frontend API call fails, but not sure if it's a parameter issue or backend issue?

  1. Enable Request mode
  2. When paused, click Edit to modify request parameters
  3. Click Send and observe the response
  4. Quickly identify the issue without changing code or recompiling
🎭 Scenario 2: Mock Backend API

Backend API not ready yet, but frontend needs to develop?

  1. Enable Request mode
  2. When paused, click Mock
  3. Enter your custom JSON response data
  4. Click Mock Response - the page receives your defined data
⚠️ Scenario 3: Test Error Handling

Need to test frontend handling of 500 errors, network timeouts, etc.?

  1. Enable Response mode
  2. When paused, click Edit
  3. Change Status Code to 500 or other error codes
  4. Click Send to verify frontend error handling logic
📝 Scenario 4: Temporarily Modify Response Data

Need to show more list items to test pagination or infinite scroll?

  1. Enable Response mode
  2. Pause the list API and click Edit
  3. Modify the JSON total count or add more data items
  4. Click Send - page immediately displays modified data

✨ Features

📊 Request List

  • Real-time Capture: Automatically records all network requests from the page
  • Type Filtering: Filter by API, JSON, JS, CSS, Image, etc.
  • Keyword Search: Search by URL, method, status code
  • Domain Grouping: Group requests by domain

📝 Request Editor

  • Visual Editing: Table-based editing for Query params, Headers, Body
  • JSON Formatting: Auto-beautify JSON with syntax highlighting
  • Multiple Body Types: JSON, Form-data, x-www-form-urlencoded, Raw

👁 Response Preview

  • JSON Tree View: Collapsible tree with search and copy
  • Image/Video Preview: Preview media files directly in the panel
  • HTML Preview: Toggle between source code and rendered preview

🔄 Request Replay

  • One-click Resend: Click Send button to resend the request
  • Modify and Resend: Change any parameters before sending for debugging

💡 Tips & Tricks

URL Pattern Matching
  • * - Intercept all requests
  • */api/* - Only intercept requests containing /api/
  • *.json - Only intercept JSON files
  • example.com/* - Only intercept specific domain
JSON Editor Shortcuts
  • Click Format button to beautify JSON
  • Click Minify button to compress JSON
  • Editor automatically detects JSON syntax errors
Preserve Log

Check Preserve log to keep request records after page navigation

⚠️ Important Notes
  • Interception uses Chrome Debugger API; a debug banner appears at the top of the browser when enabled
  • Close DevTools or toggle the Intercept switch off to disable debug mode
  • After updating the extension, close and reopen DevTools

❓ FAQ

Why is there a yellow bar at the top of the browser when interception is enabled?
This is Chrome's security notice indicating the page is being debugged. This is normal behavior and the bar disappears when interception is disabled.
Why are some requests not being intercepted?
Check: 1) Is the Intercept toggle on? 2) Is the URL pattern correct? 3) Was the request made after enabling interception?
The page is stuck after intercepting a request. What do I do?
Paused requests must be handled manually (Continue/Drop/Mock). If there are multiple paused requests, click "Continue All" in the toolbar to release all at once.
Can mock data be persisted?
The current version uses one-time mocking that requires manual operation each time. Persistent mock rules are planned for future versions.
Does it support WebSocket interception?
The current version primarily supports HTTP/HTTPS requests. WebSocket support is planned for future releases.
